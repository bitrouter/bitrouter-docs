**BitRouter Cloud** ([cloud.bitrouter.ai](https://cloud.bitrouter.ai)) is an optional hosted layer over the [open-source core](/docs/get-started/introduction). The core is identical either way — Cloud adds what needs a server you don't run yourself, and you can attach it to a **self-hosted** binary with `bitrouter auth login`. It's an account and a network, not a different product. See [Self-host vs Cloud](/docs/get-started/self-host-vs-cloud) for the decision.

## What a Cloud account adds

- **Managed marketplace** — the hosted provider network. Today that's [Managed Models](/docs/cloud/managed-models): one account, no upstream keys, billed per request, with open models served 25% below official. Managed [Tools](/docs/cloud/managed-tools) and [Agents](/docs/cloud/managed-agents) are on the way.
- **SLA** — an uptime commitment on the hosted endpoint.
- **[Workspaces & teams](/docs/cloud/workspaces)** — shared workspaces with per-member seats and access control.
- **Priority support** and **managed billing** — one wallet, billed per request; failed requests aren't billed.

## Get going

- New to the hosted service? **[Get Started](/docs/cloud/get-started)** — sign in, set up billing, make your first managed request.
- Run inference for others? **[Register as a provider](/docs/guides/register-as-a-provider)** to list your models on the network.
