Every AI agent today faces the same problem: you pick one model and send every call to it.
Choose the best model and you burn money on tasks a cheaper one could handle.
Choose a cheap model and your agent chokes on anything hard.

BitRouter fixes this. It sits between your agent and the LLM providers, watches each session in real time, and routes every call to the cheapest model that can handle it — automatically escalating to a more capable model when things go wrong.

## The cost problem in agent workloads

Agent sessions are not single prompts. A typical coding agent session might involve 20–50 LLM calls: reading files, planning changes, writing code, running tests, fixing errors. Most of those calls are straightforward — the agent is reading a file or making a small edit. Only a few require deep reasoning.

Yet today, the entire session runs on one model. If you pick Claude Opus or GPT-4o for quality, you're paying top-tier prices for every single call, including the easy ones.

Our analysis of thousands of agent sessions shows that **60–70% of calls within a session can be handled by a smaller, cheaper model** without any loss in outcome quality.

## How BitRouter works

BitRouter is a drop-in proxy. You change one line in your agent config — swap the base URL — and everything else stays the same.

```bash
# Before
OPENAI_BASE_URL=https://api.openai.com/v1

# After
OPENAI_BASE_URL=https://api.bitrouter.ai/v1
```

Behind that URL, BitRouter:

1. **Classifies** each incoming call based on the prompt content, conversation history, and task context
2. **Routes** to the cheapest model that meets the difficulty threshold
3. **Monitors** the response quality in real time
4. **Escalates** to a more capable model if the response quality drops or the agent enters an error loop

The entire routing decision adds less than 10ms of latency at p99.

## SWE-bench results

We evaluated BitRouter on [SWE-bench Verified](https://www.swebench.com/), the standard benchmark for AI coding agents. Our balanced configuration achieves:

- **93.2% resolve rate** (vs. 95.8% for all-Opus baseline)
- **42% of the cost** of running every call on the most capable model
- **87% escalation recovery rate** — when a cheaper model fails, escalation to a better model recovers the task 87% of the time

The cost-optimized configuration pushes savings even further — down to 28% of baseline cost — while maintaining a 89.1% resolve rate.

## Getting started

BitRouter works with any OpenAI-compatible agent framework. Setup takes under a minute:

```typescript
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.bitrouter.ai/v1",
  apiKey: process.env.BITROUTER_API_KEY,
});
```

That's it. Your existing agent code, prompts, and tool definitions all work unchanged.
