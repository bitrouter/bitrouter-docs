AI Agent 按需运行，使用模式不可预测。传统 SaaS 订阅、预充值额度和 API 密钥都不适合。BitRouter 提供 Agent 原生的支付基础设施——Agent 通过委托账户直接认证并按需付费，在你定义的边界内运行。

## 基于账户的访问

BitRouter 不发放静态 API 密钥，而是通过 Agent 账户认证每一个请求。支付在使用时按需处理——无预付余额、无供应商锁定。

### 支持的定价模式

- **按调用定价** — 每次 API 请求固定费用
- **按 Token 定价** — 根据输入/输出 Token 计费（适用于 LLM）

## Payment Delegation Key (PDK)

要让 Agent 代表你行动，你需要创建一个 Payment Delegation Key。PDK 授予 Agent 来自你账户的委托支付权限——而无需暴露你的凭证。你定义约束条件，Agent 在这些边界内自主运行。

### 权限参数

| 参数 | 类型 | 描述 |
|-----------|------|-------------|
| `max_transaction_value` | number | 单笔交易允许的最大金额 |
| `daily_spending_limit` | number | 24 小时内允许的总支出金额 |
| `valid_until` | timestamp | PDK 失效时间 |
| `allowed_recipients` | address[] | PDK 可以向其发送交易的服务列表 |

对于大多数使用场景，设置 `daily_spending_limit` 和 `valid_until` 即可提供足够的保护，同时允许 Agent 自主运行。

<Callout type="info">
PDK 基于策略引擎构建，在基础设施层面强制执行支出约束。如需高级配置，如服务白名单和细粒度交易规则，请参阅 [SDK 参考](/docs/sdk-reference)。
</Callout>

## 支付基础设施

在底层，BitRouter 的 Agent 原生支付由稳定币结算（USDC）驱动，运行在高性能网络上。这实现了即时、低成本、无需许可的交易——没有传统支付处理器的摩擦。

<Callout type="info">
**为什么使用稳定币？** 传统支付通道（信用卡、发票）需要人工身份验证，无法支持自主的 Agent 到服务的支付。稳定币结算消除了这些障碍，同时保持以美元计价。Agent 和开发者使用熟悉的美元金额——结算层只是一个实现细节。
</Callout>

### 支持的网络

- Base (USDC)
- Solana (USDC)

### 签名选项

对于希望不通过托管控制台直接访问的开发者，BitRouter 支持使用任何兼容钱包进行原始签名。详见 [SDK 参考](/docs/sdk-reference) 了解集成细节。
