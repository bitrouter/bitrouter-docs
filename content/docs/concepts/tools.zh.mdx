title: 工具（Tools）
description: 工具即 MCP 服务器——BitRouter 网关跨多个 host 代理工具的发现与调用，通过单一端点对外提供。
---

在 BitRouter 上，agent 的**工具**就是 [MCP](https://modelcontextprotocol.io) 服务器。每个 MCP 服务器暴露一组可调用的工具——搜索 API、数据库、文件系统、支付通道——agent 可以在运行时发现它们，并作为任务的一部分加以调用。

问题在于，这些服务器分散在各处，各有各的地址、各自的鉴权、各自的访问方式。一个想使用十个工具服务器的 agent，通常得知道十个端点。

## 为什么需要网关

BitRouter 的 **MCP 网关**位于这些服务器之前并对其进行代理。你的 agent 连接到一个 BitRouter 端点；在其背后，网关把**工具发现**和**工具调用**转发到正确的上游 host，并将响应回传。一个连接，多个工具服务器。

把工具统一经由单个网关路由，能为你带来三样原本需要为每个 agent 重复搭建的东西：

- **统一鉴权** —— agent 只需向 BitRouter 鉴权一次，而无需为每个上游工具服务器各自携带凭证。
- **发现** —— 跨 host 的工具在同一处呈现，agent 无需被预先接线到每个服务器即可发现可用项。
- **策略** —— 因为每次工具调用都经过网关，这里是统一执行规则的天然位置，而不必信任每个 agent 自觉守规。

这与 BitRouter 处理模型的方式如出一辙：你不必把 agent 指向众多供应商，而是指向一个能触达它们全部的路由器。
