Copilot's Bring-Your-Own-Key (BYOK) support lets you back its chat with any OpenAI-compatible endpoint. Point that endpoint at BitRouter and Copilot routes across the whole [registry](/docs/concepts/models).

<Callout type="warn">
**BYOK covers chat, agent, and edit — not inline code completions.** Autocomplete still runs on GitHub's own models and cannot be routed through a custom endpoint.
</Callout>

## Prerequisites

- BitRouter running — local proxy at `http://127.0.0.1:4356`, or [BitRouter Cloud](/docs/get-started/quickstart) at `https://api.bitrouter.ai`.
- A BitRouter model that supports **tool calling and streaming** (a ≥128k context window is recommended) — Copilot requires these.

## Point Copilot at BitRouter

<Tabs items={['Copilot CLI', 'VS Code']}>
<Tab value="Copilot CLI">

The CLI configures BYOK through environment variables:

```bash
export COPILOT_PROVIDER_BASE_URL=http://127.0.0.1:4356
export COPILOT_MODEL=openai/gpt-4o
export COPILOT_PROVIDER_API_KEY=local-placeholder   # any value for the local proxy
copilot
```

<Callout type="info">
If requests 404, append `/v1` to `COPILOT_PROVIDER_BASE_URL` — the CLI normally adds the route path itself.
</Callout>

</Tab>
<Tab value="VS Code">

Open the model picker in Copilot Chat → **Manage Language Models** → **Add Models**, choose the **OpenAI Compatible / Custom Endpoint** provider, and enter:

- **Base URL** — `http://127.0.0.1:4356/v1`
- **API Key** — any value for the local proxy; your BitRouter key for Cloud
- **Model** — a registry id such as `openai/gpt-4o`

<Callout type="warn">
The native OpenAI-compatible provider is currently **VS Code Insiders** only. On stable VS Code, use a community "OpenAI-compatible provider" extension.
</Callout>

</Tab>
</Tabs>

<Callout type="info">
For **Cloud**, swap the base URL to `https://api.bitrouter.ai` (CLI) or `https://api.bitrouter.ai/v1` (VS Code) and use your BitRouter key.
</Callout>

## Pick a model

Use any registry id in `provider/model` form — `openai/gpt-4o`, `anthropic/claude-sonnet-4-6`, `google/gemini-2.5-pro`, optionally with a `:cost` / `:latency` variant. See [Models](/docs/concepts/models).
