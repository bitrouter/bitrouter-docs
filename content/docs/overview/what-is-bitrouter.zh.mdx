## 什么是 BitRouter？

BitRouter 是一个专为 AI Agent 运行时打造的开源 LLM 网关。它是一个高性能 Rust 代理，让 [OpenClaw](https://openclaw.ai)、[Claude Code](https://claude.ai/claude-code)、[OpenCode](https://opencode.ai)、[Codex CLI](https://github.com/openai/codex)、[Kilo Code](https://github.com/Kilo-Org/kilocode) 等 Agent 通过单一 OpenAI 兼容端点，统一访问来自 OpenAI、Anthropic、Google 及自定义提供商的模型。

单一二进制文件部署，零基础设施依赖。Agent 在运行时自主发现并路由到 LLM、工具和子 Agent——内置安全护栏、MCP 和 A2A 网关支持，通过 CLI + TUI 实现完整的可观测性。对于不想自托管的团队，BitRouter 还提供托管服务，支持基于 x402/Solana 的 Agent 原生支付——无 KYC、无地域限制。

## 为什么选择 BitRouter？

- **Agent 原生网关** — 为自主 Agent 而非人工交互式聊天机器人而构建。Agent 在运行时自主发现和选择模型，具备 Agent 身份验证、支付委托和技能注册等功能。
- **零运维部署** — 单一 Rust 二进制文件。无需 Postgres、Redis 或 Docker 编排。`cargo install bitrouter && bitrouter` 即可启动交互式设置向导并运行服务器。
- **智能路由** — 多提供商路由，支持成本和性能优化。跨协议路由（例如 OpenAI 格式 → Anthropic 提供商），自动回退。路由开销低于 10 毫秒。
- **Agent 防火墙** — 在代理层检查、警告、脱敏或拦截风险内容。内置安全护栏无需应用层改动即可保护 Agent 免受不安全输出的影响。
- **MCP & A2A 网关** — 作为 MCP 服务器代理，让 Agent 跨主机发现和调用工具。支持 A2A v0.3.0 协议，实现 Agent 身份验证、发现和任务分派。
- **技能注册中心** — 遵循 [agentskills.io](https://agentskills.io) 标准，追踪和暴露 Agent 技能，使 Agent 能够在 BitRouter 网络上注册并服务请求。
- **Agent 原生支付** — 为 LLM、工具和 API 提供 402/MPP 支付处理。在托管服务上按请求付费——无需信用卡、预充值或发票。
- **CLI + TUI 可观测性** — 从终端实时监控和控制 Agent 会话，支持逐请求费用追踪和指标统计。

## BitRouter 与替代方案对比

| | BitRouter | OpenRouter | LiteLLM | 直接 API |
|---|---|---|---|---|
| **类型** | 开源 Rust 代理 + 托管选项 | 仅托管 SaaS（闭源） | Python 代理 + SDK（开放核心） | 提供商 SDK |
| **部署** | 单一二进制，零依赖 | 仅云端 | Docker/K8s，需要 Postgres + Redis | 按提供商分别设置 |
| **路由开销** | 低于 10ms | ~25–40ms | 中位 ~0.5ms（GIL 限制大规模吞吐量） | 不适用 |
| **支付方式** | Agent 原生 402/MPP 按使用付费 | 信用卡、加密货币充值、支付宝 | 仅 BYOK — 不处理支付 | 每个提供商单独计费 |
| **访问方式** | 无需许可——无 KYC 或地域限制 | 需要创建账户 | 自托管 | 按提供商分别注册 |
| **Agent 特性** | Agent 防火墙、MCP 网关、A2A 网关、技能注册中心、CLI/TUI 可观测性 | 工具调用、标准 API 密钥 | MCP 网关、A2A 协议网关 | 无 |
| **路由** | 跨协议路由，支持回退和升级 | 自动路由器（基于 NotDiamond）、提供商路由 | 负载均衡、回退、冷却 | 手动选择模型 |
| **可观测性** | CLI + TUI，逐请求费用追踪，内置指标 | — | 管理面板、OpenTelemetry、Prometheus | 每个提供商各自的控制台 |
| **API** | OpenAI 兼容 | OpenAI 兼容 | OpenAI 兼容 | 提供商特定 |
| **许可证** | Apache 2.0 | 闭源（SDK 为 Apache 2.0） | MIT（企业功能闭源） | 不适用 |

| Anthropic | ✅     | Messages API                     |

## 支持的 Agent 运行时

BitRouter 可作为支持自定义 API base URL 的 Agent 运行时的即插即用代理。将运行时指向 `http://localhost:8787` 即可路由到任何已配置的提供商。

| 运行时 | 集成方式 |
| ------- | ----------- |
| [OpenClaw](https://github.com/openclaw/openclaw) | [原生插件](https://github.com/bitrouter/bitrouter-openclaw) |
| [Claude Code](https://github.com/anthropics/claude-code) | CLI + Skills |
| [ZeroClaw](https://github.com/zeroclaw-labs/zeroclaw) | CLI + Skills |
| [Codex CLI](https://github.com/openai/codex) | CLI + Skills |
| [OpenCode](https://github.com/opencode-ai/opencode) | CLI + Skills |
| [Kilo Code](https://github.com/Kilo-Org/kilocode) | CLI + Skills |

任何支持自定义 OpenAI 或 Anthropic base URL 的 Agent 运行时均可开箱即用地使用 BitRouter。

## 常见问题

<Accordions type="single">
  <Accordion title="BitRouter 与 OpenRouter 或 LiteLLM 有什么不同？">
      BitRouter 专为 Agent 运行时而构建：
      - **可自托管**：与 OpenRouter（仅云端）不同，BitRouter 以单一 Rust 二进制文件交付，无需任何基础设施依赖即可在任何地方运行。
      - **Agent 原生支付**：与 LiteLLM（仅 BYOK）不同，BitRouter 的托管服务支持自主 402/MPP 按使用付费，Agent 无需人工设置计费。
      - **零运维**：与 LiteLLM（需要 Postgres + Redis）不同，BitRouter 仅需二进制文件即可运行。
      - **Agent 安全**：内置 Agent 防火墙，可在代理层检查、脱敏或拦截风险内容——OpenRouter 和 LiteLLM 均不提供此功能。
      - **高性能**：Rust 异步运行时意味着稳定的低延迟，不受 Python GIL 限制。
  </Accordion>
  <Accordion title="我可以自托管 BitRouter 还是必须使用托管服务？">
      两者皆可。BitRouter 完全开源（Apache 2.0）。你可以在 BYOK 模式下使用自己的提供商 API 密钥自托管——BitRouter 会自动检测环境变量中的密钥，无需配置文件。也可以使用 Cloud 模式，通过 x402/Solana 进行 Agent 原生支付，获得完全托管的体验。
  </Accordion>
  <Accordion title="哪些 Agent 运行时可以使用 BitRouter？">
      任何支持自定义 OpenAI 或 Anthropic base URL 的 Agent 都可以开箱即用地使用 BitRouter——只需将其指向 `http://localhost:8787`。包括 OpenClaw、Claude Code、ZeroClaw、Codex CLI、OpenCode、Kilo Code，以及任何基于 OpenAI SDK 的 Agent。请参阅集成指南了解具体设置步骤。
  </Accordion>
  <Accordion title="什么是 Agent Skills？">
      [Agent Skills](https://github.com/bitrouter/agent-skills) 赋予你的 AI Agent 在 BitRouter 网络上注册、配置服务和开始处理请求的能力。在任何支持技能的 Agent 运行时（Claude Code、Copilot、Cursor、Codex 等）中使用 `npx skills add BitRouterAI/agent-skills` 即可安装。
  </Accordion>
  <Accordion title="BitRouter 支持哪些协议？">
      BitRouter 支持 OpenAI Chat Completions API、OpenAI Responses API、Anthropic Messages API 和 Google Generative AI API。同时作为 MCP 网关实现工具发现，以及 A2A 网关（v0.3.0）实现 Agent 间通信、身份验证和任务分派。
  </Accordion>
  <Accordion title="我是模型供应商——如何接入 BitRouter？">
      我们欢迎提供商集成。如果你是有意在 BitRouter 上架模型的供应商，请[提交 Issue](https://github.com/bitrouter/bitrouter/issues) 或加入我们的 [Discord](https://discord.gg/G3zVrZDa5C) 告知你的服务。

      如果你想自行运行代理，请查看 [bitrouter 仓库](https://github.com/bitrouter/bitrouter)。
  </Accordion>
</Accordions>
