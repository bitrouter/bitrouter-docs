BitRouter Cloud is the fastest way to access 200+ models. No provider API keys to manage — BitRouter handles provider access for you. Pay per token via your on-chain wallet.

## Install & Launch

```bash
cargo install bitrouter
bitrouter
```

On first launch with no configuration, BitRouter starts the interactive setup wizard.

## Choose Your Provider

The wizard presents two options:

```
? Choose a provider mode:
  > Default (BitRouter Cloud)
    Bring Your Own Key
```

Select **Default** and press **Enter**. This configures BitRouter to route all requests through BitRouter Cloud.

<Callout type="info">
Choose **Bring Your Own Key** if you want to use your own OpenAI, Anthropic, or Google API keys instead. See the [Manual](/docs/overview/manual-guide) guide for that path.
</Callout>

## Create Your Account

The wizard opens [app.bitrouter.ai](https://app.bitrouter.ai) in your browser. Sign up using social login or by connecting an existing account.

## Fund Your Account

Add credits at [Credits](https://app.bitrouter.ai/dashboard/credits). Your funds remain in your account until the moment of use — no prepaid credit pools or minimum commitments.

## Generate a Payment Delegation Key (PDK)

1. Go to [Keys](https://app.bitrouter.ai/dashboard/keys)
2. Click "Generate PDK"
3. Configure permissions (spending limits, expiration, allowed recipients)
4. Copy your PDK (format: `br-p-[base58-encoded-key]`)

The wizard prompts you to paste your PDK:

```
? Enter your PDK: br-p-••••••••
```

<Callout type="info">
An agent authority lets AI agents pay for services on your behalf within boundaries you define. Learn more about how agent wallets work and advanced configuration options in the [Agent Wallet](/docs/overview/agent-wallet#agent-authority) guide.

For advanced use cases, you can also authenticate directly with a signer via the SDK. When using a raw signer, the base URL is `https://beta.bitrouter.ai`; when using a PDK, the base URL is `https://api.app.bitrouter.ai/v1`. See the [SDK Reference](/docs/sdk-reference) to learn more.
</Callout>

## Verify

The wizard writes your config, starts the API server, and runs a health check:

```
✓ Configuration saved to ~/.bitrouter/bitrouter.yaml
✓ API server listening on 127.0.0.1:8787
✓ Health check passed
```

<Callout type="info">
You can re-run the wizard at any time with `bitrouter init`.
</Callout>

## Making Your First Request

<Callout type="info">
Want to try it out without writing code? Experiment with models directly in the [BitRouter Playground](https://app.bitrouter.ai/playground/llms).
</Callout>

<Callout type="info">
The examples below use the chat completions endpoint for models. BitRouter also supports tools (MCP) and agents (A2A). See [Models, Tools, & Agents](/docs/overview/service-primitives) for all available service primitives and endpoints.
</Callout>

### Basic Example

```bash
curl -X POST "https://api.app.bitrouter.ai/v1/chat/completions" \
  -H "Authorization: Bearer br-p-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-5",
    "messages": [
      {"role": "user", "content": "Hello! What can you help me with?"}
    ],
    "max_tokens": 200
  }'
```

### Streaming

For real-time, token-by-token responses, add `stream: true` to your request. Responses use the Server-Sent Events format:

<Tabs items={['JavaScript', 'Python']}>
<Tab value="JavaScript">

```javascript
const response = await fetch('https://api.app.bitrouter.ai/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer br-p-[your-api-key]',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'anthropic/claude-opus-4.6',
    messages: [{ role: 'user', content: 'Tell me a short story' }],
    stream: true,
    max_tokens: 500
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const lines = decoder.decode(value).split('\n').filter(line => line.trim());
  for (const line of lines) {
    if (line.startsWith('data: ') && line.slice(6) !== '[DONE]') {
      const content = JSON.parse(line.slice(6)).choices[0]?.delta?.content;
      if (content) process.stdout.write(content);
    }
  }
}
```

</Tab>
<Tab value="Python">

```python
import requests
import json

response = requests.post(
    'https://api.app.bitrouter.ai/v1/chat/completions',
    headers={
        'Authorization': 'Bearer br-p-[your-api-key]',
        'Content-Type': 'application/json'
    },
    json={
        'model': 'openai/gpt-5',
        'messages': [{'role': 'user', 'content': 'Tell me a story'}],
        'stream': True,
        'max_tokens': 500
    },
    stream=True
)

for line in response.iter_lines():
    if line:
        line = line.decode('utf-8')
        if line.startswith('data: '):
            data = line[6:]
            if data == '[DONE]':
                break
            content = json.loads(data)['choices'][0]['delta'].get('content', '')
            if content:
                print(content, end='', flush=True)
```

</Tab>
</Tabs>

Each SSE message contains a JSON chunk with `choices[0].delta.content`. The stream ends with `data: [DONE]`.

## Request Parameters

Standard OpenAI-compatible parameters are supported:

```json
{
  "model": "openai/gpt-5",           // Required: Model identifier
  "messages": [                        // Required: Conversation messages
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Hello!"}
  ],
  "stream": false,                     // Optional: Enable streaming (default: false)
  "max_tokens": 500,                   // Optional: Maximum tokens to generate
  "temperature": 0.7,                  // Optional: Sampling temperature (0.0-2.0)
  "top_p": 1.0,                        // Optional: Nucleus sampling
  "frequency_penalty": 0.0,            // Optional: Frequency penalty
  "presence_penalty": 0.0              // Optional: Presence penalty
}
```

## Authentication & Security

Manage your PDKs and API keys at the [Dashboard](https://bitrouter.ai/dashboard) — generate new keys, monitor usage, and revoke compromised ones.

<Callout type="warn">
Store keys securely using environment variables. Never commit them to version control or expose them in client-side code. If a key is compromised, revoke it immediately from the dashboard and generate a replacement.
</Callout>

```bash
export BITROUTER_API_KEY="br-p-[your-key]"
```

## Usage & Billing

### How Costs Work

Payments are calculated based on the type of request:

- **Chat Completions**: Billed per token (input + output). Each provider sets their own per-token pricing.
- **Image Generation & Editing**: Billed per generation. Pricing varies by model and resolution.
- **Video Generation**: Billed per generation based on resolution and duration (e.g., 720p/5s vs 1080p/10s).

Use the quote endpoints to get exact price estimates before committing to a generation. See the [API Reference](/docs/api-reference) for details.

### Monitor Your Usage

Track your spending and usage at [Usage](https://app.bitrouter.ai/dashboard/usage):
- Current balance
- Usage history
- Spending by model
- Transaction history
