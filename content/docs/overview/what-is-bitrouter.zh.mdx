## 什么是 BitRouter？

BitRouter 是一个专为 AI Agent 运行时打造的开源 LLM 网关。它是一个高性能 Rust 代理，让 [OpenClaw](https://openclaw.ai)、[OpenCode](https://opencode.ai)、[Claude Code](https://claude.ai/claude-code) 等 Agent 通过单一 OpenAI 兼容端点，统一访问来自 OpenAI、Anthropic、Google 等提供商的 200+ 模型。

单一二进制文件部署，零基础设施依赖。Agent 在运行时自主发现并路由到 LLM、工具和子 Agent，通过 CLI + TUI 实现完整的可观测性。对于不想自托管的团队，BitRouter 还提供托管服务，支持 Agent 原生支付——无 KYC、无地域限制。

## 为什么选择 BitRouter？

- **Agent 原生网关** — 为自主 Agent 而非人工交互式聊天机器人而构建。Agent 在运行时自主发现和选择模型，具备 Agent 身份验证和支付委托密钥等功能。
- **零运维部署** — 单一 Rust 二进制文件。无需 Postgres、Redis 或 Docker 编排。`cargo install bitrouter && bitrouter start` 即可启动。
- **智能路由** — 可配置的路由表将模型名称映射到提供商。支持成本和性能优化，自动回退和升级。路由开销低于 10 毫秒。
- **Agent 原生支付** — 在托管服务上按请求付费。无需信用卡、预充值或发票。Agent 可自主完成支付。
- **CLI + TUI 可观测性** — 从终端实时监控和控制 Agent 会话。

## BitRouter 与 OpenRouter 对比

| | BitRouter | OpenRouter |
|---|---|---|
| **类型** | 开源 Rust 代理 + 托管选项 | 仅托管 SaaS（闭源） |
| **部署** | 自托管（单一二进制）或托管 | 仅云端 |
| **支付方式** | Agent 原生按使用付费 | 信用卡、加密货币充值、支付宝 |
| **访问方式** | 无需许可——无 KYC 或地域限制 | 需要创建账户 |
| **路由** | 可配置路由表，支持回退 | 自动路由器（基于 NotDiamond）、提供商路由 |
| **开销** | 低于 10ms | ~25–40ms |
| **Agent 特性** | Agent 身份验证、支付委托、CLI/TUI 可观测性 | 工具调用、标准 API 密钥 |
| **模型** | 200+ 模型 | 400+ 模型 / 60+ 提供商 |
| **API** | OpenAI 兼容 | OpenAI 兼容 |
| **许可证** | Apache 2.0 | 闭源（SDK 为 Apache 2.0） |

## BitRouter 与 LiteLLM 对比

| | BitRouter | LiteLLM |
|---|---|---|
| **类型** | Rust 代理（单一二进制） | Python 代理 + SDK（开放核心） |
| **部署** | 单一二进制，零依赖 | Docker/K8s，需要 Postgres + Redis |
| **性能** | 异步 Rust，无 GIL — 开销低于 10ms | 中位开销 ~0.5ms，但 Python GIL 限制大规模吞吐量 |
| **支付** | 内置 Agent 原生支付（托管服务） | 仅 BYOK — 不处理支付 |
| **Agent 特性** | Agent 身份验证、支付委托、CLI/TUI | MCP 网关、A2A 协议网关 |
| **路由** | 路由表，支持回退和升级 | 负载均衡、回退、冷却 |
| **模型** | 200+ 模型 | 2,600+ 模型 / 120+ 提供商 |
| **可观测性** | CLI + TUI，内置指标 | 管理面板、OpenTelemetry、Prometheus |
| **API** | OpenAI 兼容 | OpenAI 兼容 |
| **许可证** | Apache 2.0 | MIT（企业功能闭源） |

## BitRouter 与直接 API 访问对比

| | BitRouter | 直接 API（如 OpenAI、Anthropic） |
|---|---|---|
| **模型** | 通过一个端点访问所有提供商的 200+ 模型 | 仅该提供商的模型 |
| **计费** | 单一 Agent 账户或 BYOK | 每个提供商单独计费 |
| **路由** | 自动选择模型，支持回退 | 手动选择模型 |
| **设置** | 更改 base URL，使用现有 SDK | 每个提供商需要单独的 SDK/密钥 |
| **可观测性** | 跨所有提供商的统一 CLI/TUI | 每个提供商各自的控制台 |
| **成本** | 通过智能路由优化 | 按所选模型全价付费 |

## 常见问题

<Accordions type="single">
  <Accordion title="BitRouter 与 OpenRouter 或 LiteLLM 有什么不同？">
      BitRouter 专为 Agent 运行时而构建：
      - **可自托管**：与 OpenRouter（仅云端）不同，BitRouter 以单一 Rust 二进制文件交付，无需任何基础设施依赖即可在任何地方运行。
      - **Agent 原生支付**：与 LiteLLM（仅 BYOK）不同，BitRouter 的托管服务支持自主按使用付费，Agent 无需人工设置计费。
      - **零运维**：与 LiteLLM（需要 Postgres + Redis）不同，BitRouter 仅需二进制文件即可运行。
      - **高性能**：Rust 异步运行时意味着稳定的低延迟，不受 Python GIL 限制。
  </Accordion>
  <Accordion title="我可以自托管 BitRouter 还是必须使用托管服务？">
      两者皆可。BitRouter 完全开源（Apache 2.0）。你可以使用自己的提供商 API 密钥自托管，也可以使用带 Agent 原生支付的托管服务获得完全托管的体验。
  </Accordion>
  <Accordion title="哪些 Agent 运行时可以使用 BitRouter？">
      任何支持 OpenAI API 的 Agent 都可以开箱即用地使用 BitRouter——只需更改 base URL。包括 OpenClaw、OpenCode、Claude Code，以及任何基于 OpenAI SDK 的 Agent。请参阅集成指南了解具体设置步骤。
  </Accordion>
  <Accordion title="我是模型供应商——如何接入 BitRouter？">
      我们尚未开放公开的供应商接入，但非常期待你的来信。如果你是有意在 BitRouter 上架模型的供应商，请通过 [contact@bitrouter.ai](mailto:contact@bitrouter.ai) 联系我们，或加入我们的 [Discord](https://discord.gg/MkeG9Zwuaw) 告知你的服务。

      如果你想自行运行代理，请查看 [bitrouter 仓库](https://github.com/bitrouter/bitrouter)。
  </Accordion>
</Accordions>
