**BitRouter Cloud** ([cloud.bitrouter.ai](https://cloud.bitrouter.ai)) is an optional hosted layer over the [open-source core](/docs/get-started/introduction). The core is identical either way — Cloud adds what needs a server you don't run yourself, and you can attach it to a **self-hosted** binary with `bitrouter auth login`. It's an account and a network, not a different product. See the [capability comparison](#self-hosted-vs-cloud) below for the decision.

## What a Cloud account adds

- **Managed marketplace** — the hosted provider network. Today that's [Managed Models](/docs/cloud/managed-models): one account, no upstream keys, billed per request, with open models served 25% below official. Managed [Tools](/docs/cloud/managed-tools) and [Agents](/docs/cloud/managed-agents) are on the way.
- **SLA** — an uptime commitment on the hosted endpoint.
- **[Workspaces & teams](/docs/cloud/workspaces)** — shared workspaces with per-member seats and access control.
- **Priority support** and **managed billing** — one wallet, billed per request; failed requests aren't billed.

None of these gate the core. You can run the open-source binary self-hosted and attach a Cloud account to it whenever you want the hosted network — the binary doesn't change.

## Self-hosted vs Cloud

The core is **identical** whether you self-host the binary or use the hosted service. Every routing, fallback, model-variant, BYOK, local-model, guardrail, observability, MCP, ACP, and structured-output capability works exactly the same either way. The `—` rows below are commercial and network features, not core inference capabilities.

| Capability | Self-hosted | Cloud |
| --- | --- | --- |
| Universal API + cross-protocol routing | ✅ | ✅ |
| BYOK (bring your own provider keys) | ✅ | ✅ |
| Local / private model serving | ✅ | ✅ |
| Model fallback & provider selection | ✅ | ✅ |
| Model variants & presets | ✅ | ✅ |
| Guardrails | ✅ | ✅ |
| Observability (OTLP trace + metric export) | ✅ | ✅ |
| MCP & ACP gateways | ✅ | ✅ |
| Structured outputs | ✅ | ✅ |
| SLA | — | ✅ |
| Team seats & roles | — | ✅ |
| Priority support | — | ✅ |
| Managed billing | — | ✅ |
| Hosted provider network & discounts | — | ✅ |
| Agentic payment marketplace | — | ✅ |
