**BitRouter Cloud**（[cloud.bitrouter.ai](https://cloud.bitrouter.ai)）是[开源核心](/docs/get-started/introduction)之上的可选托管层。两种方式下核心完全相同——Cloud 只是补上那些需要由我们托管的服务器才能提供的能力，并且你可以用 `bitrouter auth login` 把它附加到**自托管**的二进制上。它是一个账户与网络，而非另一套产品。决策参见下方的能力对比表。

## Cloud 账户带来什么

- **托管市场** — 即托管的供应商网络。目前是[托管模型](/docs/cloud/managed-models)：一个账户、无需上游密钥、按请求计费，开放模型比官方价低 25%。托管[工具](/docs/cloud/managed-tools)与[Agents](/docs/cloud/managed-agents)即将推出。
- **SLA** — 对托管端点的可用性承诺。
- **[工作区与团队](/docs/cloud/workspaces)** — 带每成员席位与访问控制的共享工作区。
- **优先支持**与**托管计费** — 一个钱包，按请求计费；失败的请求不计费。

以上都不会限制核心能力。你完全可以自托管开源二进制，并在需要托管网络时随时附加一个 Cloud 账户——二进制本身不会改变。

## 自托管 vs 云

无论你自托管二进制还是使用托管服务，核心都**完全相同**。所有路由、回退、模型变体、BYOK、本地模型、防护、可观测性、MCP、ACP 与结构化输出能力的表现都一模一样。下表中的 `—` 行是商业与网络功能，而非核心推理能力。

| 能力 | 自托管 | 云 |
| --- | --- | --- |
| 通用 API + 跨协议路由 | ✅ | ✅ |
| BYOK（自带供应商密钥） | ✅ | ✅ |
| 本地 / 私有模型部署 | ✅ | ✅ |
| 模型回退与供应商选择 | ✅ | ✅ |
| 模型变体与预设 | ✅ | ✅ |
| 防护（Guardrails） | ✅ | ✅ |
| 可观测性（OTLP 追踪 + 指标导出） | ✅ | ✅ |
| MCP 与 ACP 网关 | ✅ | ✅ |
| 结构化输出 | ✅ | ✅ |
| SLA | — | ✅ |
| 团队席位与角色 | — | ✅ |
| 优先支持 | — | ✅ |
| 托管计费 | — | ✅ |
| 托管供应商网络与折扣 | — | ✅ |
| Agent 自主支付市场 | — | ✅ |

## 开始使用

- 第一次使用托管服务？**[Get Started](/docs/cloud/get-started)** — 登录、设置计费、发出你的第一个托管请求。
- 想为他人提供推理？**[注册成为供应商](/docs/guides/register-as-a-provider)**，把你的模型上架到网络。
