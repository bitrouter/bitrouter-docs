# @bitrouter/provider

Vercel AI SDK integration for BitRouter. Wraps `@ai-sdk/openai` with SIWx-authenticated fetch for seamless use with AI SDK functions. Uses `LanguageModelV3` (AI SDK v4+).

```bash
npm install @bitrouter/provider ai
```

**Peer dependencies:** `@bitrouter/client`, `ai`, `@ai-sdk/openai`, `@ai-sdk/provider`

## bitrouter(options)

Creates an AI SDK-compatible provider backed by SIWx-authenticated fetch.

```typescript
import { bitrouter } from "@bitrouter/provider";

const br =bitrouter({
  signer,
  baseURL: "https://beta.bitrouter.ai",
});
```

### BitRouterOptions

| Property | Type | Required | Default | Description |
|----------|------|----------|---------|-------------|
| `signer` | `ClientSigner` | Yes | — | Wallet signer |
| `baseURL` | `string` | No | `"https://beta.bitrouter.ai"` | API server URL |
| `fetch` | `typeof fetch` | No | — | Custom fetch implementation |
| `siwxDomain` | `string` | No | — | Override SIWx signing domain |
| `apiKey` | `string` | No | `"siwx"` | Dummy API key (real auth is SIWx) |

## br.chat(modelId)

Returns a `LanguageModelV3` instance for use with AI SDK functions.

**Parameters:**
- `modelId: string` — Model identifier (e.g., `"openai/gpt-4o"`, `"anthropic/claude-sonnet-4"`)

**Returns:** `LanguageModelV3`

## Usage

### Generate Text

```typescript
import { bitrouter } from "@bitrouter/provider";
import { generateText } from "ai";

const br =bitrouter({ signer });

const result = await generateText({
  model: br.chat("openai/gpt-4o"),
  prompt: "What is BitRouter?",
});

console.log(result.text);
```

### Stream Text

```typescript
import { streamText } from "ai";

const stream = await streamText({
  model: br.chat("anthropic/claude-sonnet-4"),
  prompt: "Tell me a story",
});

for await (const chunk of stream.textStream) {
  process.stdout.write(chunk);
}
```

### Multi-Turn Conversation

```typescript
import { generateText } from "ai";

const result = await generateText({
  model: br.chat("openai/gpt-4o"),
  messages: [
    { role: "system", content: "You are a helpful assistant." },
    { role: "user", content: "What is BitRouter?" },
  ],
});
```

### Using Different Models

All models are accessed through the same OpenAI-compatible interface:

```typescript
const br =bitrouter({ signer });

// OpenAI
const gpt = br.chat("openai/gpt-4o");

// Anthropic
const claude = br.chat("anthropic/claude-sonnet-4");

// Google
const gemini = br.chat("google/gemini-2.0-flash");
```

## How It Works

Under the hood, `@bitrouter/provider` wraps `@ai-sdk/openai`'s `createOpenAI()` with [`wrapFetchWithSigner`](/docs/sdk-reference/ts-client#wrapfetchwithsigner) for automatic SIWx authentication and x402 payments. This means it supports all features of the OpenAI-compatible provider, including tool calls, structured output, and streaming.
