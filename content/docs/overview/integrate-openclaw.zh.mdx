BitRouter 以原生插件的形式接入 OpenClaw。安装插件、配置服务商，所有 Agent 请求即刻通过本地多服务商代理路由 — 开箱即用的故障转移、负载均衡和统一 API 访问。

## 安装插件

```bash
openclaw plugins install @bitrouter/openclaw-plugin
```

安装时，插件会在 `127.0.0.1:8787` 启动 BitRouter 守护进程，将 `bitrouter` 注册为 OpenClaw 的服务商，并开始路由符合条件的模型请求。

## 路由 Agent 请求

默认情况下，BitRouter 仅拦截有显式路由规则的模型。若要让所有请求 — 包括未命名路由的请求 — 都经由 BitRouter 转发，在 OpenClaw 设置中开启 `interceptAllModels`：

```jsonc
{
  "plugins": {
    "entries": {
      "bitrouter": {
        "enabled": true,
        "config": {
          "interceptAllModels": true
        }
      }
    }
  }
}
```

<Callout type="info">
即使未定义命名路由，也可以使用 `"provider:model_id"` 语法（例如 `"openai:gpt-4o"`），BitRouter 将直接路由该请求。
</Callout>

### 定义虚拟模型

定义 `fast`、`smart` 等别名，映射到多个服务商端点并配置路由策略：

```jsonc
{
  "plugins": {
    "entries": {
      "bitrouter": {
        "enabled": true,
        "config": {
          "models": {
            "fast": {
              "strategy": "load_balance",
              "endpoints": [
                { "provider": "openai", "modelId": "gpt-4o-mini" },
                { "provider": "anthropic", "modelId": "claude-3.5-haiku" }
              ]
            },
            "smart": {
              "strategy": "priority",
              "endpoints": [
                { "provider": "anthropic", "modelId": "claude-sonnet-4-20250514" },
                { "provider": "openai", "modelId": "gpt-4o" }
              ]
            }
          }
        }
      }
    }
  }
}
```

策略：`priority`（默认 — 按顺序尝试，失败时故障转移）与 `load_balance`（轮询）。

## 验证

```bash
bitrouter status
```

你应该看到守护进程状态为 **healthy**、监听地址以及已注册的服务商。Agent 也可以在运行时通过 `bitrouter_status` 工具做同样的检查。

## 启用的能力

- **故障转移与负载均衡** — 按优先级或轮询方式跨服务商路由。[了解更多](/docs/overview/runtime-optimization)
- **虚拟模型** — 定义 `fast`、`smart` 等别名，映射到多个服务商端点并配置路由策略
- **自定义服务商** — 基于 `openai`、`anthropic` 或 `google` 派生，几行配置即可接入 OpenRouter 等服务商
- **用量统计** — 按模型、按服务商的请求追踪
- **Agent 密钥管理** — 通过 `bitrouter_keygen`、`bitrouter_account`、`bitrouter_keys` 等 Agent 工具管理 Ed25519 密钥对与 JWT
