## 主目录

BitRouter 按以下顺序解析工作目录：

1. 命令行指定 `--home-dir <PATH>`
2. 当前目录（如果存在 `./bitrouter.yaml`）
3. `BITROUTER_HOME` 环境变量（如果指向已存在的目录）
4. `~/.bitrouter`（默认回退 — 自动创建）

### 目录结构

```text
<home>/
├── bitrouter.yaml   # 主配置文件
├── .env             # 环境变量（自动加载）
├── .gitignore       # 忽略 logs/、run/、.env
├── .keys/           # 本地 Ed25519 账户密钥和 JWT
├── logs/            # 服务器日志
└── run/             # PID 文件和运行时状态
```

## 设置向导

创建配置最简单的方式：

```bash
bitrouter init
```

向导提供两种路径：

- **Cloud** — 通过 x402 支付连接 BitRouter 云节点（需要 Solana 钱包）
- **BYOK** — 使用你自己的 OpenAI、Anthropic、Google 或自定义提供商 API 密钥

向导会为你生成 `bitrouter.yaml` 和 `.env`。

## 配置文件

主配置文件为 `bitrouter.yaml`。完整示例：

```yaml
server:
  listen: 127.0.0.1:8787
  log_level: info

database:
  url: "sqlite://${BITROUTER_HOME}/bitrouter.db?mode=rwc"

providers:
  openai:
    api_key: ${OPENAI_API_KEY}
  anthropic:
    api_key: ${ANTHROPIC_API_KEY}

models:
  default:
    strategy: priority
    endpoints:
      - provider: openai
        model_id: gpt-4o
  fast:
    strategy: load_balance
    endpoints:
      - provider: openai
        model_id: gpt-4o-mini
      - provider: anthropic
        model_id: claude-haiku-4-5-20251001

guardrails:
  enabled: true
  upgoing:
    api_keys: redact
    private_keys: block
```

### 服务器

| 字段 | 默认值 | 描述 |
|------|--------|------|
| `listen` | `127.0.0.1:8787` | API 服务器地址和端口 |
| `log_level` | `info` | 日志级别 |
| `control.socket` | `bitrouter.sock` | 守护进程控制 Unix 套接字 |

### 数据库

| 字段 | 默认值 | 描述 |
|------|--------|------|
| `url` | 主目录中的 SQLite | 连接 URL — 支持 `sqlite://`、`postgres://`、`mysql://` |

## 环境变量

BitRouter 自动加载 `<home>/.env`，然后读取进程环境变量。在 `bitrouter.yaml` 中使用 `${VAR}` 占位符：

```yaml
providers:
  openai:
    api_key: ${OPENAI_API_KEY}
    api_base: ${OPENAI_BASE_URL}
```

### `.env` 文件

```bash
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
GOOGLE_API_KEY=AIza...
```

<Callout type="warn">
`.env` 文件可能包含敏感信息。默认的 `.gitignore` 已将其排除 — 请勿提交到版本控制。
</Callout>

## 零配置模式

如果环境中已有提供商 API 密钥，BitRouter 会自动检测并启用直接路由，无需配置文件：

```bash
export OPENAI_API_KEY=sk-...
bitrouter serve
```

然后使用 `openai:gpt-4o` 作为模型名称。自动检测使用每个内置提供商定义的 `env_prefix`（如 `OPENAI` 映射 `OPENAI_API_KEY` 和 `OPENAI_BASE_URL`）。

## CLI 覆盖

每个路径和数据库 URL 都可以按次覆盖：

```bash
bitrouter serve \
  --home-dir /opt/bitrouter \
  --config-file /etc/bitrouter/bitrouter.yaml \
  --env-file /etc/bitrouter/.env \
  --run-dir /var/run/bitrouter \
  --logs-dir /var/log/bitrouter \
  --db "postgres://user:pass@localhost/bitrouter"
```

这些标志在每个子命令上都可用。
