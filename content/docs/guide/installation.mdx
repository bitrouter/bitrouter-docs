## Prerequisites

- [Rust toolchain](https://rustup.rs/) (stable, 1.80+)

## Install from crates.io

The fastest way to get BitRouter running:

```bash
cargo install bitrouter
```

This builds and installs the `bitrouter` binary into `~/.cargo/bin`.

## Install from source

Clone the repository and build with all default features:

```bash
git clone https://github.com/bitrouter/bitrouter.git
cd bitrouter
cargo build --release
```

The binary is at `target/release/bitrouter`. Copy it to your PATH or run it directly.

### Feature flags

BitRouter supports optional compile-time features:

| Feature | Default | Description |
|---------|---------|-------------|
| `tui` | Yes | Interactive terminal UI |
| `swig` | No | On-chain Swig wallet operations (requires Solana dependencies) |

To build without the TUI:

```bash
cargo build --release --no-default-features
```

To build with Swig wallet support:

```bash
cargo build --release --features swig
```

## Verify the installation

```bash
bitrouter status
```

This prints the resolved home directory, listen address, configured providers, and daemon status.

## First run

Launch BitRouter with no arguments:

```bash
bitrouter
```

On first run with no providers configured, the interactive setup wizard starts automatically. It walks you through provider selection, API key entry, and generates your `bitrouter.yaml`. After setup, the TUI and API server start together.

To skip the TUI and run a foreground API server:

```bash
bitrouter serve
```

To run as a background daemon:

```bash
bitrouter start
```

<Callout type="info">
If you already have provider API keys in your environment (e.g. `OPENAI_API_KEY`), BitRouter auto-detects them and enables direct routing without any configuration file. See [Configuration](/docs/guide/configuration) for details.
</Callout>

## Update

To update to the latest release:

```bash
cargo install bitrouter
```

BitRouter also checks for new releases automatically and prints a notice after commands finish when an update is available.
