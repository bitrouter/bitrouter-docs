BitRouter 的核心是一个高性能 Rust 代理，将 Agent 请求路由到最优的服务提供商。可配置的路由表、自动故障转移、低于 10ms 的路由开销——无需任何基础设施依赖。

## 路由表

路由表将模型名称映射到服务提供商。你可以定义哪些提供商服务哪些模型，并设置优先级排序和故障转移链。

```yaml
# 路由配置示例
routes:
  - model: "gpt-4o"
    providers:
      - name: openai
        priority: 1
      - name: azure-openai
        priority: 2  # 备用
  - model: "claude-sonnet-4-20250514"
    providers:
      - name: anthropic
        priority: 1
```

当请求到达时，BitRouter 将模型名称与路由表匹配，并将其转发到最高优先级的可用提供商。

## 故障转移与升级

当提供商不可用或返回错误时，BitRouter 会自动转移到链中的下一个提供商。这一过程对 Agent 透明——Agent 无需处理重试或提供商特定的逻辑。

- **提供商故障转移** — 在 5xx 错误或超时时自动重试下一个提供商
- **模型升级** — 当主模型无法处理请求时，可选择路由到更强大的模型

## 性能

BitRouter 使用异步 Rust 构建，零基础设施依赖——无需 Postgres、Redis 或 Docker 编排。这使路由开销始终保持在 10ms 以下。
