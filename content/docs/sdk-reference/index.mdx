# BitRouter SDKs

<Callout type="warn">
The BitRouter SDKs are in **early beta** and under active development. APIs, interfaces, and behavior may change without notice. Pin to a specific version and watch the changelog for breaking changes.
</Callout>

Official SDKs for interacting with [BitRouter](https://bitrouter.ai) â€” a decentralized AI inference marketplace. Both SDKs provide wallet-based authentication (SIWx), automatic micropayment handling (x402), and OpenAI-compatible chat completions.

| SDK | Language | Package | Status |

## Installation

### TypeScript

```bash
# Core client (required)
npm install @bitrouter/client

# Chain-specific signer (pick one or both)
npm install @bitrouter/svm   # Solana
npm install @bitrouter/evm   # Ethereum / Base

# Vercel AI SDK integration (optional)
npm install @bitrouter/provider ai
```

### Python

```bash
# Core + Solana support
pip install "bitrouter[solana]"

# Core + EVM support
pip install "bitrouter[evm]"

# With OpenAI SDK integration
pip install "bitrouter[openai]"

# Everything
pip install "bitrouter[all]"
```

## Quick Comparison

Both SDKs share the same architecture and API surface:

| Feature | TypeScript | Python |
|---------|-----------|--------|
| Core client | `BitRouterClient` | `BitRouterClient` (async) |
| Solana signer | `SvmClientSigner` | `SvmClientSigner` |
| EVM signer | `EvmClientSigner` | `EvmClientSigner` |
| AI SDK integration | `@bitrouter/provider` (Vercel AI SDK) | `create_openai_client()` (OpenAI SDK) |
| Auth | SIWx (automatic) | SIWx (automatic) |
| Payments | x402 (automatic) | x402 (automatic) |
| HTTP | `fetch` / custom fetch | `httpx` / custom transport |
| Streaming | SSE via `Response.body` | SSE via `httpx.Response` |

## Authentication

Both SDKs use **SIWx (Sign-In-With-X)** for wallet-based authentication. Every request is automatically signed with your wallet, providing permissionless access without API keys.

## Payments

Both SDKs integrate with **x402** for automatic micropayment handling. When API requests require payment, the SDK handles it transparently via your wallet.
