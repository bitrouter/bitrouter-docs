BitRouter 实现了 [A2A 协议](https://google.github.io/A2A/)（v0.3.0），允许代理通过标准化接口发现其他代理、与之通信并委派任务。`bitrouter-a2a` crate 是独立的，不依赖 `bitrouter-core`。

## 配置上游代理

在 `bitrouter.yaml` 中注册外部 A2A 代理：

```yaml
a2a_agents:
  - name: "research-agent"
    url: "https://research.example.com"
    headers:
      Authorization: "Bearer ${RESEARCH_TOKEN}"

  - name: "search-agent"
    url: "https://search.example.com"
    card_path: "/custom/card.json"  # 默认为 /.well-known/agent-card.json
```

| 字段 | 必填 | 描述 |
|------|------|------|
| `name` | 是 | 用于路由和计价的标识符 |
| `url` | 是 | 上游 A2A 代理的基础 URL |
| `headers` | 否 | 额外的 HTTP 头（如认证令牌） |
| `card_path` | 否 | 自定义代理卡片端点路径 |

## 支持的方法

### JSON-RPC（POST /a2a）

所有 A2A JSON-RPC 方法通过单一端点提供：

| 方法 | 描述 |
|------|------|
| `message/send` | 向代理发送消息 |
| `message/stream` | 流式消息响应（SSE） |
| `tasks/get` | 获取任务状态和历史 |
| `tasks/list` | 列出任务 |
| `tasks/cancel` | 取消正在运行的任务 |
| `tasks/resubscribe` | 重新订阅任务更新（SSE） |
| `agent/getAuthenticatedExtendedCard` | 获取已认证的代理卡片 |
| `tasks/pushNotificationConfig/set` | 设置推送通知配置 |
| `tasks/pushNotificationConfig/get` | 获取推送通知配置 |
| `tasks/pushNotificationConfig/list` | 列出推送通知配置 |
| `tasks/pushNotificationConfig/delete` | 删除推送通知配置 |

### REST 端点

BitRouter 也提供 REST 等效接口：

| 方法 | 路径 | 描述 |
|------|------|------|
| `GET` | `/.well-known/agent-card.json` | 标准代理卡片发现 |
| `GET` | `/card` | 代理卡片别名 |
| `GET` | `/extendedAgentCard` | 扩展代理卡片 |
| `POST` | `/message:send` | 发送消息 |
| `POST` | `/message:stream` | 流式消息（SSE） |
| `GET` | `/tasks/{id}` | 获取特定任务 |
| `GET` | `/tasks` | 列出所有任务 |
| `POST` | `/tasks/{id}:cancel` | 取消任务 |
| `POST` | `/tasks/{id}:subscribe` | 订阅任务更新（SSE） |

## 代理卡片发现

BitRouter 在 `/.well-known/agent-card.json` 发布自己的代理卡片。上游代理通过从配置的 URL（使用 `card_path` 或默认的 `/.well-known/agent-card.json`）获取卡片来自动发现。

## 计价

为 402 支付计量设置每个代理的价格：

```yaml
a2a_agent_pricing:
  research-agent:
    default_cost_per_call: 0.01
    methods:
      "message/send": 0.05
      "tasks/get": 0.01
  search-agent:
    default_cost_per_call: 0.005
```

详见 [402 支付](/docs/guide/402-payment)了解计价如何与支付层集成。

## 示例：发送消息

```bash
curl -X POST http://localhost:8787/a2a \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "message/send",
    "params": {
      "message": {
        "role": "user",
        "parts": [{"kind": "text", "text": "总结最近的机器学习论文"}]
      }
    }
  }'
```

<Callout type="info">
传输协议为 HTTP（JSON-RPC + REST）并支持 SSE 流式传输。目前不支持 gRPC。
</Callout>
