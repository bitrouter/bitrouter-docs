在你自己的基础设施上运行开源的 BitRouter 代理，通过 BitRouter Network 路由 AI 模型请求。自托管让你完全掌控部署、模型配置和基础设施。

## 概述

BitRouter 代理是一个开源服务，将你的 AI 模型端点连接到 BitRouter Network 路由器。你配置要暴露的模型、设定定价并部署代理——路由器负责发现、认证和请求路由。

**源代码：** [github.com/AIMOverse/bitrouter-proxy](https://github.com/AIMOverse/bitrouter-proxy)

## 前置要求

- 一个 Solana 或 EVM 钱包
- 一个或多个 OpenAI 兼容的 AI 模型 API 端点（自有或第三方密钥）
- Docker（推荐）或 Rust 工具链

## 第 1 步：安装 CLI 并生成密钥对

```bash
# Install CLI
curl -fsSL https://aimo-cli-releases.s3.ap-northeast-1.amazonaws.com/install.sh | sh

# Generate keypair
aimo keygen

# Keypair saved to ~/.config/aimo/keypair.json

# Check your address
aimo address
```

你的密钥对用于向路由器进行认证和签署配置更新。

<Callout type="warn">
请妥善保管你的密钥对。此文件可控制你的代理注册。
</Callout>

## 第 2 步：确保 API 兼容性

你的模型端点必须与 OpenAI 的 Chat Completion API 格式兼容。

### 要求

你的 API 必须：
- 接受 `/v1/chat/completions` 的 POST 请求
- 支持 OpenAI 格式的 JSON 请求
- 返回符合 OpenAI 结构的响应
- **在每个响应中包含 usage 数据**

### 请求格式

```json
POST /v1/chat/completions
Content-Type: application/json

{
  "model": "your-model-name",
  "messages": [
    {"role": "user", "content": "Hello, AI!"}
  ],
  "stream": false,
  "max_tokens": 150,
  "temperature": 0.7
}
```

### 响应格式（非流式）

```json
{
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "Hello! How can I help you today?"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 15,
    "completion_tokens": 8,
    "total_tokens": 23
  }
}
```

### 响应格式（流式）

```http
Content-Type: text/event-stream

data: {"choices":[{"delta":{"content":"Hello"}}]}

data: {"choices":[{"delta":{"content":" there!"}}]}

data: {"choices":[{"delta":{},"finish_reason":"stop"}],"usage":{"prompt_tokens":15,"completion_tokens":8,"total_tokens":23}}

data: [DONE]
```

<Callout type="warn">
**重要：必须包含 Usage 数据**

你必须在每个响应中包含 `usage` 对象。没有 usage 数据，Token 计数和计费将无法正常工作。

**非流式：** 在最终响应对象中包含 `usage`。
**流式：** 在 `[DONE]` 之前的最后一个 chunk 中包含 `usage`。
</Callout>

### 适配非 OpenAI API

如果你的 API 使用不同的格式，可以创建一个轻量级适配器：

**Python 示例：**
```python
from flask import Flask, request, jsonify

@app.route('/v1/chat/completions', methods=['POST'])
def openai_compatible():
    data = request.json
    messages = data['messages']

    # Call your native API
    response = your_model.generate(messages)

    # Convert to OpenAI format
    return jsonify({
        'choices': [{
            'message': {
                'role': 'assistant',
                'content': response.text
            },
            'finish_reason': 'stop'
        }],
        'usage': {
            'prompt_tokens': response.prompt_tokens,
            'completion_tokens': response.completion_tokens,
            'total_tokens': response.total_tokens
        }
    })
```

**常用 API 网关：**
- [LiteLLM](https://github.com/BerriAI/litellm) - 支持 100+ LLM API 的代理
- 使用你偏好的语言编写自定义中间件

## 第 3 步：创建配置文件

创建 `proxy.toml` 文件来定义你的模型和连接设置。

### 配置示例

```toml
[router]
url = "https://beta.aimo.network"

# SIWx wallet-based authentication
[router.siwx]
chain_id = "solana:5eykt4UsFv8P8NJdTREpY1vzqKqZKvdp"
keypair_file = "~/.config/aimo/keypair.json"

# Model 1: OpenAI GPT-4o
[[models]]
name = "gpt-4o"
display_name = "OpenAI: GPT-4o"
provider_name = "openai"
context_length = 128000
input_modalities = ["text"]
output_modalities = ["text"]

[models.pricing]
# USDC per million tokens
prompt = 2.5
completion = 10.0

[models.params]
model = "gpt-4o"
api_base = "https://api.openai.com/v1"
api_key = "std::env::OPENAI_API_KEY"

# Model 2: DeepSeek
[[models]]
name = "deepseek-chat-v3"
display_name = "DeepSeek: DeepSeek V3"
provider_name = "deepseek"
context_length = 64000

[models.pricing]
prompt = 0.5
completion = 1.0

[models.params]
model = "deepseek-chat"
api_base = "https://api.deepseek.com/v1"
api_key = "std::env::DEEPSEEK_API_KEY"

# Model 3: Google Gemini
[[models]]
name = "gemini-2.5-flash"
display_name = "Google: Gemini 2.5 Flash Preview"
provider_name = "google"
context_length = 1000000

[models.pricing]
prompt = 0.3
completion = 0.85

[models.params]
model = "gemini-2.5-flash"
api_base = "https://generativelanguage.googleapis.com/v1"
api_key = "std::env::GOOGLE_API_KEY"
```

<Callout type="info">
**单个或多个模型**：每个模型对应一个 `[[models]]` 块。按需添加即可。
</Callout>

### 配置字段参考

#### 路由器设置

| 字段 | 描述 | 必填 |
|-------|-------------|----------|
| `url` | BitRouter 路由器端点 | 是 |
| `siwx.chain_id` | 钱包的 CAIP-2 链 ID | 是 |
| `siwx.keypair_file` | 密钥对文件路径 | 是（Solana） |
| `siwx.private_key` | 私钥（文件的替代方式） | 是（EVM） |

#### 模型元数据

| 字段 | 描述 | 必填 | 示例 |
|-------|-------------|----------|---------|
| `name` | 唯一模型标识符 | 是 | `gpt-4o` |
| `display_name` | 面向用户的名称 | 是 | `OpenAI: GPT-4o` |
| `provider_name` | 你的供应商标识 | 是 | `openai` |
| `context_length` | 最大上下文窗口 | 否 | `128000` |
| `input_modalities` | 支持的输入类型 | 否 | `["text", "image"]` |
| `output_modalities` | 支持的输出类型 | 否 | `["text"]` |

#### 定价

| 字段 | 描述 | 单位 | 必填 |
|-------|-------------|------|----------|
| `prompt` | 输入 Token 成本 | USDC / 百万 Token | 是 |
| `completion` | 输出 Token 成本 | USDC / 百万 Token | 是 |
| `image` | 图像输入成本 | USDC / 张 | 否 |

#### 模型参数

| 字段 | 描述 | 必填 |
|-------|-------------|----------|
| `model` | API 的模型标识符 | 是 |
| `api_base` | API 端点的 Base URL | 是 |
| `api_key` | 认证密钥 | 是 |

### 多模态模型配置

对于支持图像输入的模型：

```toml
[[models]]
name = "gpt-4-vision"
display_name = "OpenAI: GPT-4 Vision"
provider_name = "openai"
input_modalities = ["text", "image"]
output_modalities = ["text"]

[models.pricing]
prompt = 10.0
completion = 30.0
image = 1.0  # Per image cost

[models.params]
model = "gpt-4-vision-preview"
api_base = "https://api.openai.com/v1"
api_key = "std::env::OPENAI_API_KEY"
```

### 环境变量

将你的 API 密钥设置为环境变量：

```bash
export OPENAI_API_KEY="sk-your-openai-key"
export DEEPSEEK_API_KEY="sk-your-deepseek-key"
export GOOGLE_API_KEY="your-google-api-key"
```

在 `proxy.toml` 中，使用 `std::env::` 前缀引用它们：

```toml
api_key = "std::env::OPENAI_API_KEY"
```

## 第 4 步：部署代理

### 方案 A：Docker（推荐）

创建 `compose.yml`：

```yaml
services:
  proxy:
    image: ghcr.io/aimoverse/bitrouter-proxy:main
    container_name: bitrouter-proxy
    network_mode: host
    volumes:
      - ./proxy.toml:/etc/proxy/proxy.toml
      - ~/.config/aimo/keypair.json:/root/.config/aimo/keypair.json:ro
    environment:
      CONFIG_FILE_PATH: /etc/proxy/proxy.toml
      OPENAI_API_KEY: ${OPENAI_API_KEY}
      DEEPSEEK_API_KEY: ${DEEPSEEK_API_KEY}
      GOOGLE_API_KEY: ${GOOGLE_API_KEY}
```

```bash
# Start proxy in background
docker compose up -d

# View logs
docker compose logs -f

# Stop proxy
docker compose down
```

### 方案 B：从源码构建

```bash
# Install proxy via Cargo
cargo install bitrouter-proxy

# Run proxy with your config
bitrouter-proxy --config proxy.toml

# Or with verbose logging
bitrouter-proxy --config proxy.toml --log-level debug
```

### 验证连接

当你看到以下日志时，说明代理已成功运行：

```
[INFO] Connected to router: https://beta.aimo.network
[INFO] Model registered: gpt-4o
[INFO] Model registered: deepseek-chat-v3
[INFO] Ready to serve requests
```

<Callout type="info">
代理会在路由器重启时自动重连。这在活跃开发期间是正常现象。
</Callout>

### 配置验证

部署前测试你的配置：

```bash
# Dry-run to validate config
bitrouter-proxy --config proxy.toml --validate

# Run with verbose logging
bitrouter-proxy --config proxy.toml --log-level debug
```

## 测试你的部署

### 测试 API 兼容性

```bash
# Test non-streaming
curl https://your-api.example.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "your-model",
    "messages": [{"role": "user", "content": "Hello!"}],
    "stream": false
  }'
```

**验证响应包含：**
- 正确的 JSON 结构
- 包含 `message` 对象的 `choices` 数组
- **包含 Token 计数的 `usage` 对象**
- `finish_reason` 字段

### 测试流式

```bash
curl https://your-api.example.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "your-model",
    "messages": [{"role": "user", "content": "Hello!"}],
    "stream": true
  }'
```

**验证：**
- `Content-Type: text/event-stream`
- 每行以 `data: ` 开头
- **最后一个 chunk 包含 `usage`**
- 以 `data: [DONE]` 结尾

## 常见问题

### 缺少 Usage 数据
**问题：** 响应缺少 `usage` 字段
**解决方案：** 始终计算并在响应中包含 Token 计数

### 流式格式不正确
**问题：** 流式 chunk 格式错误
**解决方案：** 每个 chunk 必须是以 `data: ` 为前缀的有效 JSON

### Content-Type 错误
**问题：** 流式返回 `application/json`
**解决方案：** 流式响应使用 `text/event-stream`

### 代理连接失败
**问题：** 无法连接到路由器
**解决方案：** 检查密钥对文件是否有效以及网络连接

### 模型未注册
**问题：** 模型未出现在网络中
**解决方案：** 检查配置语法，重启代理，查看日志

### Token 计数精度
**问题：** Token 计数不准确
**解决方案：** 为你的模型使用正确的 Tokenizer（例如 GPT 模型使用 tiktoken）
