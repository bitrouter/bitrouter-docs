无论你自托管二进制还是使用托管服务，BitRouter 的核心都**完全一致**。所有路由、故障转移、模型变体、BYOK、本地模型、护栏、可观测性、MCP、ACP 以及结构化输出能力，两种方式下的表现一模一样。开源二进制采用 Apache 2.0 协议、免费——自带密钥或运行本地模型，除了你自己的供应商费用之外不欠任何费用。

云端账号并不是另一款产品。它是一个账号和一张供应商网络，你可以用 `bitrouter auth login` 把它挂到**任意**二进制上——包括自托管的二进制。它所增加的是运营与商业层面的内容，而非被门禁的推理功能。

## 云端账号额外提供什么

- **SLA** — 对托管端点的可用性承诺。
- **团队席位与角色** — 带逐成员访问控制的共享工作区。
- **优先支持** — 直接的支持渠道。
- **托管计费** — 单一钱包，按请求计费；失败的请求不计费。
- **托管供应商网络** — [托管供应商](/docs/cloud/managed-models)（一个账号，无需上游密钥）、[折扣模型](/docs/cloud/managed-models)（开源模型 25% 折扣）以及 [agentic 支付](/docs/cloud/payment)（x402/MPP 按用量付费市场）。

这些都不会对核心设门禁。你完全可以自托管运行开源二进制，并在任何想用托管网络的时候把云端账号挂上去——二进制本身不会改变。

## 对比

| 能力 | 自托管 | 云端 |
| --- | --- | --- |
| 通用 API + 跨协议路由 | ✅ | ✅ |
| BYOK（自带供应商密钥） | ✅ | ✅ |
| 本地 / 私有模型服务 | ✅ | ✅ |
| 模型故障转移与供应商选择 | ✅ | ✅ |
| 模型变体与预设 | ✅ | ✅ |
| 护栏 | ✅ | ✅ |
| 可观测性（OTLP trace + 指标导出） | ✅ | ✅ |
| MCP 与 ACP 网关 | ✅ | ✅ |
| 结构化输出 | ✅ | ✅ |
| SLA | — | ✅ |
| 团队席位与角色 | — | ✅ |
| 优先支持 | — | ✅ |
| 托管计费 | — | ✅ |
| 托管供应商网络与折扣 | — | ✅ |
| agentic 支付市场 | — | ✅ |

标记 `—` 的行是商业与网络功能，而非核心推理能力——后者全部为 `✅`/`✅`。
