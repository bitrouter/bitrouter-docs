Welcome to BitRouter — the open-source LLM gateway for AI agent runtimes. Access 200+ models from OpenAI, Anthropic, Google, and more through a single OpenAI-compatible endpoint.

## Quick Start

Get started in 3 simple steps and make your first API call in under 5 minutes.

### Step 1: Create Your Account

Visit [Console](https://console.bitrouter.ai) and sign up by connecting an external wallet (Base or Solana) or using social login, which automatically creates an embedded wallet for you.

### Step 2: Fund Your Account

Deposit USDC into your self-custodial wallet at [Credits](https://console.bitrouter.ai/dashboard/credits). Your funds remain in your own wallet until the moment of use.

### Step 3: Generate a Payment Delegation Key (PDK)

1. Go to [Keys](https://console.bitrouter.ai/dashboard/keys)
2. Click "Generate PDK"
3. Configure permissions (spending limits, expiration, allowed recipients)
4. Copy your PDK (format: `br-p-[base58-encoded-key]`)

<Callout type="info">
A PDK lets AI agents pay for services on your behalf without giving them full control of your wallet. Learn more about how PDKs work and advanced configuration options in the [Agent-Native Control](/docs/overview/agent-native-control#payment-delegation-key-pdk) guide.

PDK is optional — for fully permissionless access, you can use BitRouter directly with any EVM or Solana signer. We recommend using the [bitrouter-sdk](/docs/sdk-reference) for the best experience. When using a raw signer, the base URL is `https://beta.bitrouter.ai`; when using a PDK, the base URL is `https://api.console.bitrouter.ai/v1`. See the [SDK Reference](/docs/sdk-reference) to learn more.
</Callout>


## Making Your First Request

<Callout type="info">
Want to try it out without writing code? Experiment with models directly in the [BitRouter Playground](https://console.bitrouter.ai/playground/llms).
</Callout>

<Callout type="info">
The examples below use the chat completions endpoint for models. BitRouter also supports tools (MCP) and agents (A2A). See [Models, Tools, & Agents](/docs/overview/service-primitives) for all available service primitives and endpoints.
</Callout>

### Basic Example

```bash
curl -X POST "https://api.console.bitrouter.ai/v1/chat/completions" \
  -H "Authorization: Bearer br-p-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-5",
    "messages": [
      {"role": "user", "content": "Hello! What can you help me with?"}
    ],
    "max_tokens": 200
  }'
```

### Streaming

For real-time, token-by-token responses, add `stream: true` to your request. Responses use the Server-Sent Events format:

<Tabs items={['JavaScript', 'Python']}>
<Tab value="JavaScript">

```javascript
const response = await fetch('https://api.console.bitrouter.ai/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer br-p-[your-api-key]',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    model: 'anthropic/claude-opus-4.6',
    messages: [{ role: 'user', content: 'Tell me a short story' }],
    stream: true,
    max_tokens: 500
  })
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const lines = decoder.decode(value).split('\n').filter(line => line.trim());
  for (const line of lines) {
    if (line.startsWith('data: ') && line.slice(6) !== '[DONE]') {
      const content = JSON.parse(line.slice(6)).choices[0]?.delta?.content;
      if (content) process.stdout.write(content);
    }
  }
}
```

</Tab>
<Tab value="Python">

```python
import requests
import json

response = requests.post(
    'https://api.console.bitrouter.ai/v1/chat/completions',
    headers={
        'Authorization': 'Bearer br-p-[your-api-key]',
        'Content-Type': 'application/json'
    },
    json={
        'model': 'openai/gpt-5',
        'messages': [{'role': 'user', 'content': 'Tell me a story'}],
        'stream': True,
        'max_tokens': 500
    },
    stream=True
)

for line in response.iter_lines():
    if line:
        line = line.decode('utf-8')
        if line.startswith('data: '):
            data = line[6:]
            if data == '[DONE]':
                break
            content = json.loads(data)['choices'][0]['delta'].get('content', '')
            if content:
                print(content, end='', flush=True)
```

</Tab>
</Tabs>

Each SSE message contains a JSON chunk with `choices[0].delta.content`. The stream ends with `data: [DONE]`.

## Multi-Modal Generation

BitRouter supports image generation, image editing, and video generation alongside chat completions.

<Callout type="info">
The examples below show the most common parameters. For full parameter documentation, see the [API Reference](/docs/api-reference).
</Callout>

### Generating Images

Use `POST /image/generate` to create images from a text prompt. By default, the response returns a JSON object with base64-encoded data URLs.

<Tabs items={['curl', 'JavaScript', 'Python']}>

```bash
  -H "Content-Type: application/json" \
  -d '{
    "model": "dall-e-3",
    "prompt": "A futuristic cityscape at sunset",
    "width": "1024",
    "height": "1024",
    "output_format": "png"
  }'
```

Set `"output_type": "file"` to receive the raw binary image directly instead of JSON. File mode only supports a single image (`n = 1`).

### Editing Images

Use `POST /image/edit` to edit or inpaint an existing image. Provide the base image as a base64 data URL, and optionally include a mask for inpainting (transparent regions are edited, opaque regions are kept).

```bash
curl -X POST "https://api.console.bitrouter.ai/v1/image/edit" \
  -H "Authorization: Bearer br-p-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "flux-fill",
    "images": [{ "b64": "data:image/png;base64,..." }],
    "mask": { "b64": "data:image/png;base64,..." },
    "prompt": "Replace the sky with a starry night"
  }'
```

See the [Image Edit API Reference](/docs/api-reference/editImage) for full parameter details.

### Generating Videos

Video generation is **asynchronous**. You submit a job, poll for progress, and retrieve the result when ready.

**Step 1 — Queue a job:**

```bash
curl -X POST "https://api.console.bitrouter.ai/v1/video/queue" \
  -H "Authorization: Bearer br-p-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-v2",
    "prompt": "A drone flyover of a mountain landscape",
    "duration": "5s",
    "resolution": "720p",
    "aspect_ratio": "16:9"
  }'
```

The response returns a `task_id` you'll use to check status:

```json
{
  "id": "<task_id>",
  "status": "queued",
  "variants_expected": 1
}
```

**Step 2 — Poll for results:**

```bash
curl -X POST "https://api.console.bitrouter.ai/v1/video/retrieve" \
  -H "Authorization: Bearer br-p-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{
    "task_id": "<task_id>"
  }'
```

While generating, the response includes `status`, `progress` (0.0–1.0), and `estimated_seconds`. Once complete, the raw MP4 binary is returned directly.

**Step 3 — Clean up:**

After downloading the video, signal completion to free server-side resources:

```bash
curl -X POST "https://api.console.bitrouter.ai/v1/video/complete" \
  -H "Authorization: Bearer br-p-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "kling-v2",
    "queue_id": "<task_id>"
  }'
```

See the [Video API Reference](/docs/api-reference/queueVideo) for all parameters including image-to-video, reference images, and audio generation.

### Estimating Costs

Use the quote endpoints to get price estimates before committing to a generation:

```bash
# Image generation quote
curl -X POST "https://api.console.bitrouter.ai/v1/image/generate/quote" \
  -H "Authorization: Bearer br-p-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{ "model": "dall-e-3", "width": 1024, "height": 1024 }'

# Video generation quote
curl -X POST "https://api.console.bitrouter.ai/v1/video/quote" \
  -H "Authorization: Bearer br-p-[your-api-key]" \
  -H "Content-Type: application/json" \
  -d '{ "model": "kling-v2", "duration": "5s", "resolution": "720p" }'
```

Both return `{ "price_usd": <amount> }` with the estimated cost in USD.

<Tab value="Permissionless (Wallet)">

```typescript
import { BitRouterClient } from "@bitrouter/client";
import { SvmClientSigner, SOLANA_MAINNET_CHAIN_ID } from "@bitrouter/svm";
import { createKeyPairSignerFromBytes } from "@solana/kit";
import bs58 from "bs58";

// Load your wallet keypair
const privateKeyBytes = bs58.decode(process.env.SOLANA_PRIVATE_KEY);
const keypairSigner = await createKeyPairSignerFromBytes(privateKeyBytes);

// Create signer
const signer = new SvmClientSigner({
  signer: keypairSigner,
  chainId: SOLANA_MAINNET_CHAIN_ID,
});

// Create client (permissionless router)
const client = new BitRouterClient({
  signer,
  baseUrl: "https://beta.bitrouter.ai",
});

// Check balance
const balance = await client.sessionBalance();
console.log(`Balance: ${balance.balance_usd}`);

// Make chat completion
const response = await client.chatCompletions({
  model: "openai/gpt-5",
  messages: [{ role: "user", content: "Hello!" }],
});

const data = await response.json();
console.log(data.choices[0].message.content);

// Streaming
const streamResponse = await client.chatCompletions({
  model: "openai/gpt-5",
  messages: [{ role: "user", content: "Tell me a story" }],
  stream: true,
});

const reader = streamResponse.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;

  const chunk = decoder.decode(value);
  const lines = chunk.split('\n').filter(line => line.trim());

  for (const line of lines) {
    if (line.startsWith('data: ') && line.slice(6) !== '[DONE]') {
      const data = JSON.parse(line.slice(6));
      const content = data.choices[0]?.delta?.content;
      if (content) {
        process.stdout.write(content);
      }
    }
  }
}
```

## Available Models

### Discovering Models

**List all models via API:**

```bash
curl "https://api.console.bitrouter.ai/v1/models" \
  -H "Authorization: Bearer br-p-[your-api-key]"
```

Filter by type using the `type` query parameter (`chat`, `image`, `image_edit`, `video`, or `all`):

```bash
# List only image generation models
curl "https://api.console.bitrouter.ai/v1/models?type=image" \
  -H "Authorization: Bearer br-p-[your-api-key]"

# List only video generation models
curl "https://api.console.bitrouter.ai/v1/models?type=video" \
  -H "Authorization: Bearer br-p-[your-api-key]"
```

**Browse visually:**

Visit [Marketplace](https://console.bitrouter.ai/marketplace) to explore all available models with their capabilities and pricing.

### Model Specification

Specify models in two ways:

**1. Intelligent Routing (Recommended):**

```json
{
  "model": "openai/gpt-5"
}
```

The system automatically routes to the best available provider.

**2. Direct Provider:**

```json
{
  "model": "provider_pubkey:model_name"
}
```

Route directly to a specific provider. Find provider public keys at the [marketplace](https://bitrouter.ai/marketplace).

## Request Parameters

Standard OpenAI-compatible parameters are supported:

```json
{
  "model": "openai/gpt-5",           // Required: Model identifier
  "messages": [                        // Required: Conversation messages
    {"role": "system", "content": "You are a helpful assistant."},
    {"role": "user", "content": "Hello!"}
  ],
  "stream": false,                     // Optional: Enable streaming (default: false)
  "max_tokens": 500,                   // Optional: Maximum tokens to generate
  "temperature": 0.7,                  // Optional: Sampling temperature (0.0-2.0)
  "top_p": 1.0,                        // Optional: Nucleus sampling
  "frequency_penalty": 0.0,            // Optional: Frequency penalty
  "presence_penalty": 0.0              // Optional: Presence penalty
}
```

## Authentication & Security

Manage your PDKs and API keys at the [Dashboard](https://bitrouter.ai/dashboard) — generate new keys, monitor usage, and revoke compromised ones.

<Callout type="warn">
Store keys securely using environment variables. Never commit them to version control or expose them in client-side code. If a key is compromised, revoke it immediately from the dashboard and generate a replacement.
</Callout>

```bash
export BITROUTER_API_KEY="br-p-[your-key]"
```

## Usage & Billing

### How Costs Work

Payments are calculated based on the type of request:

- **Chat Completions**: Billed per token (input + output). Each provider sets their own per-token pricing.
- **Image Generation & Editing**: Billed per generation. Pricing varies by model and resolution.
- **Video Generation**: Billed per generation based on resolution and duration (e.g., 720p/5s vs 1080p/10s).

Use the [quote endpoints](#estimating-costs) to get exact price estimates before committing to a generation.

### Monitor Your Usage

Track your spending and usage at [Usage](https://console.bitrouter.ai/dashboard/usage):
- Current balance
- Usage history
- Spending by model
- Transaction history

### Funding Your Account

Add USDC to your wallet through:
- **Apple Pay / Google Pay** - Buy USDC directly from your mobile wallet
- **Bank Transfer** - Fund via bank transfer
- **Wallet Transfer** - Send USDC from any Solana or Base wallet

<Callout type="info">
BitRouter service fees do not include on/off ramp costs. Any fees from funding methods (e.g., card processing, bank transfer) are separate.
</Callout>

**Supported Networks:**
- Solana Mainnet (USDC)
- Base Mainnet (USDC)
