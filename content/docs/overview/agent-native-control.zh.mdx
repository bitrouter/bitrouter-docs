AI Agent 按需运行，使用模式不可预测。传统 SaaS 订阅、预充值额度和 API 密钥都不适合。BitRouter 用钱包签名取代 API 密钥——Agent 通过自托管钱包直接认证并按需付费，在你定义的边界内运行。

## 基于钱包的访问

BitRouter 不发放 API 密钥，而是通过钱包签名认证每一个请求。支付通过 [x402 协议](https://www.x402.org/) 在 Base 或 Solana 上按需处理——无托管中介、无预付余额、无供应商锁定。

### 支持的定价模式

- **按调用定价** — 每次 API 请求固定费用
- **按 Token 定价** — 根据输入/输出 Token 计费（适用于 LLM）

## Payment Delegation Key (PDK)

要让 Agent 代表你行动，你需要创建一个 Payment Delegation Key。PDK 授予 Agent 来自你钱包的委托支付权限——而无需暴露你的私钥。你定义约束条件，Agent 在这些边界内自主运行。

### 权限参数

| 参数 | 类型 | 描述 |
|-----------|------|-------------|
| `max_transaction_value` | number | 单笔交易允许的最大金额 |
| `daily_spending_limit` | number | 24 小时内允许的总支出金额 |
| `valid_until` | timestamp | PDK 失效时间 |
| `allowed_recipients` | address[] | PDK 可以向其发送交易的地址列表 |

对于大多数使用场景，设置 `daily_spending_limit` 和 `valid_until` 即可提供足够的保护，同时允许 Agent 自主运行。

<Callout type="info">
当前 PDK 实现基于 [Privy 的策略引擎](https://docs.privy.io/controls/overview)构建，采用自托管模式。未来版本将支持 signer 兼容，允许直接使用任何外部钱包。如需高级配置，如合约白名单、calldata 限制和类型化数据签名规则，请参阅 [Privy 文档](https://docs.privy.io/controls/policies/overview)。
</Callout>
