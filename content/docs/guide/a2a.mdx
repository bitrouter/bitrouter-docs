BitRouter implements the [A2A protocol](https://google.github.io/A2A/) (v0.3.0), allowing agents to discover, communicate with, and delegate tasks to other agents through a standardized interface. The `bitrouter-a2a` crate is standalone — it has no dependency on `bitrouter-core`.

## Configure upstream agents

Register external A2A agents in `bitrouter.yaml`:

```yaml
a2a_agents:
  - name: "research-agent"
    url: "https://research.example.com"
    headers:
      Authorization: "Bearer ${RESEARCH_TOKEN}"

  - name: "search-agent"
    url: "https://search.example.com"
    card_path: "/custom/card.json"  # defaults to /.well-known/agent-card.json
```

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Identifier used in routing and pricing |
| `url` | Yes | Base URL of the upstream A2A agent |
| `headers` | No | Extra HTTP headers (e.g. auth tokens) |
| `card_path` | No | Custom agent card endpoint path |

## Supported methods

### JSON-RPC (POST /a2a)

All A2A JSON-RPC methods are served at a single endpoint:

| Method | Description |
|--------|-------------|
| `message/send` | Send a message to an agent |
| `message/stream` | Stream a message response (SSE) |
| `tasks/get` | Get task status and history |
| `tasks/list` | List tasks |
| `tasks/cancel` | Cancel a running task |
| `tasks/resubscribe` | Resubscribe to task updates (SSE) |
| `agent/getAuthenticatedExtendedCard` | Get the authenticated agent card |
| `tasks/pushNotificationConfig/set` | Set push notification config |
| `tasks/pushNotificationConfig/get` | Get push notification config |
| `tasks/pushNotificationConfig/list` | List push notification configs |
| `tasks/pushNotificationConfig/delete` | Delete push notification config |

### REST endpoints

BitRouter also exposes REST equivalents:

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/.well-known/agent-card.json` | Standard agent card discovery |
| `GET` | `/card` | Alias for agent card |
| `GET` | `/extendedAgentCard` | Extended agent card |
| `POST` | `/message:send` | Send a message |
| `POST` | `/message:stream` | Stream a message (SSE) |
| `GET` | `/tasks/{id}` | Get a specific task |
| `GET` | `/tasks` | List all tasks |
| `POST` | `/tasks/{id}:cancel` | Cancel a task |
| `POST` | `/tasks/{id}:subscribe` | Subscribe to task updates (SSE) |

## Agent card discovery

BitRouter publishes its own agent card at `/.well-known/agent-card.json`. Upstream agents are discovered automatically by fetching their card from the configured URL (using `card_path` or the default `/.well-known/agent-card.json`).

## Pricing

Set per-agent pricing for 402 payment metering:

```yaml
a2a_agent_pricing:
  research-agent:
    default_cost_per_call: 0.01
    methods:
      "message/send": 0.05
      "tasks/get": 0.01
  search-agent:
    default_cost_per_call: 0.005
```

See [402 Payment](/docs/guide/402-payment) for how pricing integrates with the payment layer.

## Example: send a message

```bash
curl -X POST http://localhost:8787/a2a \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": 1,
    "method": "message/send",
    "params": {
      "message": {
        "role": "user",
        "parts": [{"kind": "text", "text": "Summarize recent ML papers"}]
      }
    }
  }'
```

<Callout type="info">
Transport is HTTP (JSON-RPC + REST) with SSE for streaming. gRPC is not currently supported.
</Callout>
