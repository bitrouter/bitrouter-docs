BitRouter supports the [Agent Skills](https://agentskills.io) open standard for registering prompt-level knowledge that agents can discover and use. Skills are `SKILL.md` files — structured prompts injected into LLM context — distinct from [MCP](/docs/guide/mcp) tools which provide runtime tool connectivity.

## Skills vs MCP tools

| | Skills | MCP Tools |
|---|--------|-----------|
| **What** | Prompt-level knowledge (SKILL.md) | Runtime tool calls (JSON-RPC) |
| **When** | Injected into LLM context before inference | Called by the LLM during inference |
| **Discovery** | `GET /v1/tools` (unified) | `GET /v1/tools` (unified) |

Both share a unified `ToolEntry` type, so skills and MCP tools appear together in the tool discovery endpoint.

## Configure skills

Register skills in `bitrouter.yaml`:

```yaml
skills:
  - name: "code-review"
    description: "Reviews code for quality, security issues, and best practices"
    source: "https://github.com/anthropics/skills"
    required_apis:
      - provider: anthropic

  - name: "translate"
    description: "Translates text between languages using LLM-powered translation"
```

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Unique skill identifier |
| `description` | Yes | Human-readable description |
| `source` | No | URL or path to the skill source |
| `required_apis` | No | Provider dependencies for the skill |

The `required_apis` field declares which providers a skill depends on, allowing BitRouter to handle payment (via [402](/docs/guide/402-payment)) transparently.

## Skills API

Manage skills at runtime through the CRUD API:

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/v1/skills` | Register a new skill |
| `GET` | `/v1/skills` | List all registered skills |
| `GET` | `/v1/skills/:name` | Get a specific skill |
| `DELETE` | `/v1/skills/:name` | Delete a skill |

Config-defined skills appear in `GET /v1/tools` immediately. Skills registered via the API are persisted in the database.

## Install skills via CLI

Skills can also be installed using the `agent-skills` npm package:

```bash
npx skills add BitRouterAI/agent-skills
```

## Unified tool discovery

All skills and MCP tools are discoverable through a single endpoint:

```bash
curl http://localhost:8787/v1/tools
```

Example response:

```json
[
  {
    "name": "code-review",
    "type": "skill",
    "description": "Reviews code for quality, security issues, and best practices"
  },
  {
    "name": "github:search_repositories",
    "type": "mcp_tool",
    "description": "Search GitHub repositories"
  }
]
```

<Callout type="info">
Skills registered in the config file are loaded on startup. Skills added via the `/v1/skills` API are stored in the database and persist across restarts.
</Callout>
