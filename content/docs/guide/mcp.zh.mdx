BitRouter 作为 MCP 代理，通过单一网关将你的代理连接到 MCP 工具服务器。`bitrouter-mcp` crate 使用纯 serde 结构体实现了 [MCP 2025-11-25](https://modelcontextprotocol.io/) 规范。

## 配置 MCP 服务器

在 `bitrouter.yaml` 中添加 MCP 服务器：

```yaml
mcp_servers:
  - name: "github"
    transport:
      type: stdio
      command: "npx"
      args: ["-y", "@modelcontextprotocol/server-github"]
      env:
        GITHUB_TOKEN: "${GITHUB_TOKEN}"

  - name: "remote-tools"
    transport:
      type: http
      url: "http://localhost:3000/mcp"
      headers:
        Authorization: "Bearer ${TOOLS_TOKEN}"
```

### 传输类型

| 类型 | 描述 |
|------|------|
| `stdio` | 启动本地进程，通过 stdin/stdout 通信 |
| `http` | 通过 HTTP+SSE 连接远程 MCP 服务器 |

### Stdio 传输字段

| 字段 | 必填 | 描述 |
|------|------|------|
| `command` | 是 | 要执行的命令 |
| `args` | 否 | 命令参数 |
| `env` | 否 | 传递给进程的环境变量 |

### HTTP 传输字段

| 字段 | 必填 | 描述 |
|------|------|------|
| `url` | 是 | MCP 服务器 URL |
| `headers` | 否 | 额外的 HTTP 头 |

## 工具过滤

限制 MCP 服务器暴露的工具：

```yaml
mcp_servers:
  - name: "github"
    transport:
      type: stdio
      command: "npx"
      args: ["-y", "@modelcontextprotocol/server-github"]
      env:
        GITHUB_TOKEN: "${GITHUB_TOKEN}"
    tool_filter:
      allow: ["search_repositories", "get_file_contents"]
```

只有 `allow` 列表中的工具会通过 BitRouter 暴露。未在列表中的工具对消费者不可见。

## 服务器分组

将 MCP 服务器组织为命名分组以进行访问控制：

```yaml
mcp_groups:
  dev_tools: ["github", "jira"]
  comms: ["slack", "email"]
```

分组可在认证策略中引用，以控制哪些账户可以访问哪些工具集。

## 支持的方法

BitRouter 代理以下 MCP 客户端到服务器方法：

| 方法 | 描述 |
|------|------|
| `initialize` | 初始化 MCP 会话 |
| `ping` | 健康检查 |
| `tools/list` | 列出可用工具 |
| `tools/call` | 调用工具 |
| `resources/list` | 列出资源 |
| `resources/read` | 读取资源 |
| `resources/templates/list` | 列出资源模板 |
| `resources/subscribe` | 订阅资源更新 |
| `resources/unsubscribe` | 取消订阅更新 |
| `prompts/list` | 列出可用提示 |
| `prompts/get` | 获取特定提示 |
| `logging/setLevel` | 设置服务器日志级别 |
| `completion/complete` | 请求补全 |

### 端点

| 方法 | 路径 | 描述 |
|------|------|------|
| `POST` | `/mcp` | JSON-RPC 请求 |
| `GET` | `/mcp/sse` | 服务器推送事件流 |

## 计价

为 402 支付计量设置每个服务器和每个工具的价格：

```yaml
mcp_server_pricing:
  github:
    default_cost_per_call: 0.001
    tools:
      search_repositories: 0.005
      expensive_tool: 0.05
```

详见 [402 支付](/docs/guide/402-payment)了解计价如何与支付层集成。

## 统一工具发现

所有 MCP 工具（和[技能](/docs/guide/skills)）通过单一发现端点暴露：

```bash
curl http://localhost:8787/v1/tools
```

这将返回所有已配置 MCP 服务器和已注册技能的统一工具列表。

<Callout type="info">
尚未支持：`tasks/*`、`sampling/createMessage`、`roots/list`、`elicitation/create` 和 Streamable HTTP 传输。这些在路线图中。
</Callout>
